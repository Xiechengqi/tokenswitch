ネットワークの仕組み

プロバイダーはローカルで cc-switch を実行。Router がトラフィックをルーティング。Market が利用者と接続——キーはプロバイダー端末から出ません。

  1. 1

    共有。 cc-switch を実行し、最寄りの Router へ SSH リバーストンネルで接続。

  2. 2

    ルーティング。 公開リクエストは Router のサブドメインに到達し、トンネル経由でプロキシ。

  3. 3

    収益化。 Token Market（従量）または Share Market（期間）に出品して収益を得る。

トラストセンター

TokenSwitch がプロバイダー、利用者、プラットフォーム境界をどう保護するか。

プロバイダー

  • ◆上流 API キーは端末から出ません。
  • ◆Share はメタデータのみ同期——Router はサブドメインと ACL のみ保存。
  • ◆Token Market 収益は ledger 記録、出金は監査可能。

利用者

  • ◆Token Market API キーはマーケット発行、プロバイダーキーとは別。
  • ◆利用量はダッシュボードで可視化・照合可能。
  • ◆Share Market：オーナーが支払い確認後にアクセス付与。

プラットフォーム境界

  • ◆Share Market は資金を預からず、送金を検証せず、返金を保証しません。
  • ◆Router はルーティングと ACL の信頼境界——決済処理者ではありません。
  • ◆Token Market は複式 ledger、Admin は残高を黙って変更できません。
